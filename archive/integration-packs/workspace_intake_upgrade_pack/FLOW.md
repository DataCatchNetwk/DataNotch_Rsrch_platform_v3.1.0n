# Workspace Intake Flow

## Intake is not a static list

The Workspace page should not only show:

```text
Workspace name
Owner
Datasets
Reports
```

It should become the entry point for research assets.

## Correct flow

```text
Create / Open Workspace
      ↓
Upload ZIP / Folder / CSV / XLSX / JSON / Parquet
      ↓
Safe storage
      ↓
ZIP extraction
      ↓
Workspace file tree
      ↓
File metadata indexing
      ↓
Dataset candidate detection
      ↓
Register selected files as raw datasets
      ↓
Dataset Registry
      ↓
Data Profiling
      ↓
Cleaning & Wrangling
      ↓
Harmonization
      ↓
Feature Engineering
      ↓
Quality Validation
      ↓
Dataset Versioning
      ↓
Research Studio
      ↓
Analytics & AI
      ↓
Outputs
```

## Page sections

1. Summary KPIs
2. Workspace list
3. Intake command center
4. Upload zone
5. File explorer
6. Dataset candidates
7. Operational actions
8. Pipeline handoff
9. Recent intake events
