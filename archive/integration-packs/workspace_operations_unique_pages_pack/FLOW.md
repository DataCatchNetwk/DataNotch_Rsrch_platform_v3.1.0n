# Workspace Operations Flow

## 1. Workspaces
Own the full research container. Users create workspace, assign datasets, create projects, and hand off work to every major platform area.

Actions:
- Create workspace
- Open workspace
- Assign dataset
- Create project
- Create task
- Handoff to Data Management, Data Preparation, Research Studio, Analytics, Outputs, Governance, System

## 2. Projects
Own objectives, milestones, timelines, and deliverables. Projects do not clean data or run models directly; they generate tasks and launch workflows.

Actions:
- Create project
- Add milestone
- Link dataset
- Launch cohort design
- Create analysis job
- Create publication deliverable

## 3. Tasks
Own operational accountability. Tasks cut across all platform stages.

Statuses:
- To Do
- In Progress
- Review
- Completed

Actions:
- Create task
- Assign owner
- Move status
- Attach dataset/project/analysis/report
- Send to reviewer queue

## 4. Runtime Monitoring
Own pipeline observability and controls.

Actions:
- Pause
- Resume
- Retry
- Cancel
- Open run details

Stages:
INGEST -> PROFILE -> CLEAN -> HARMONIZE -> FEATURE -> VALIDATE -> DESIGN -> ANALYZE -> VISUALIZE -> REPORT -> GOVERN -> EXPORT
